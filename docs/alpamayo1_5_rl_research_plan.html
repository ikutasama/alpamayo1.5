Alpamayo 1.5 强化学习科研计划

面向 ICLR/AAAI 级别投稿的改进方向：把 Chain-of-Causation 推理、离散轨迹 token 和连续 Flow Matching action expert 放进一个可验证、可消融、可实现的联合 RL 框架。

0. 一句话结论

最建议主线：Causal Flow-GRPO。 现有仓库的 RL 只优化 VLM 生成的 CoC 文本和离散轨迹 token，reward 主要由 ADE 与 comfort 组成；而 Alpamayo 1.5 的核心卖点恰恰是 CoC reasoning + diffusion/flow action expert。论文创新点可以放在“联合信用分配”：让 CoC 解释、离散 token 轨迹、连续 flow trajectory 三者共享同一个可验证 reward，却在不同 token/模块上接受不同的优势函数。

Alpamayo 1.5 模型卡明确写到模型基于 Cosmos-Reason2 VLM 和 diffusion-based trajectory decoder，输出包括 CoC reasoning traces 与 6.4 秒未来轨迹；轨迹内部表示为 unicycle model 下的 acceleration/curvature 动态 action。也就是说，这个模型天然不是“只会吐 token 的 LLM”，而是一个有可解释推理和连续控制头的 VLA。

3. 创新 Idea 排序

1Causal Flow-GRPO：CoC 与 Flow Action 的联合信用分配

核心想法：把一次 rollout 的 reward 拆成四个分量：CoC 可验证性、离散轨迹准确性、连续 flow 轨迹质量、CoC-action 一致性。训练时不再把一个标量 advantage 粗暴施加给所有 completion token，而是按 token 类型路由 advantage：CoC token 主要吃 reasoning reward，轨迹 token 主要吃 trajectory reward，二者共同吃 consistency reward。

R = w_r R_coc + w_d R_discrete_traj + w_f R_flow_traj + w_c R_consistency - w_k KL(pi || pi_ref) A_token = CoC tokens: norm(R_coc + R_consistency) traj tokens: norm(R_discrete_traj + R_consistency) flow expert: norm(R_flow_traj + R_consistency)

为什么新：VLA-R1 强调 reasoning RLVR，但不处理 Alpamayo 1.5 这种 continuous flow expert；FPO/π_RL 处理 flow policy RL，但通常不把可解释 CoC 当作需要被 causal credit assignment 约束的中间决策变量。这个方向把两条线接起来。

代码落点：先在 data_packer.py 输出 token masks，再在 trainer.py 构造 per-token advantage map；reward 分量由 aggregated_reward.py 返回，并进入 rollout metadata。

2Counterfactual CoC Reward：因果解释必须能改变动作

核心想法：对模型生成的 CoC 做结构化解析：主体、风险、因果触发、决策意图。构造轻量 counterfactual，例如“前车 cut-in 不存在”“导航要求右转变为直行”“行人不在本车道”。如果 CoC 声称某因素是决策原因，那么去掉/替换该因素后，轨迹应该出现方向、速度或安全距离上的可解释变化。

R_cf = sim(Delta action, expected_delta(causal_edit)) - penalty(spurious_cause) spurious_cause: CoC mentions a causal factor that has no measurable action effect

为什么新：普通 LLM judge 容易奖励“说得像”，但自动驾驶需要“说的原因真的驱动了规划”。这个 reward 可以把可解释性从文字质量推进到行为因果性，适合长尾驾驶场景。

代码落点：新增 finetune/rl/rewards/coc_reward.py 与 causal_consistency_reward.py。第一版不用在线调用大模型，可以先用规则、轨迹变化和已有 reference 字段做 verifiable reward。

3Flow Policy Optimization for Alpamayo：让 action expert 进入 RL

核心想法：在 stage2 SFT 之后，让 diffusion/flow action expert 接受 reward fine-tuning。由于 flow matching 的 action likelihood 难精确计算，可借鉴 FPO：用 conditional flow matching loss 的变化近似 policy ratio，做 clipped surrogate objective，并保留 SFT/参考模型 KL 或 MSE anchor。

ratio_proxy = exp(-L_flow(theta; x, t, noise) + L_flow(theta_old; x, t, noise)) L_FPO = - min(ratio_proxy * A, clip(ratio_proxy, 1-eps, 1+eps) * A) + beta_ref * ||v_theta - v_ref||^2

为什么新：当前 Alpamayo RL 路径主要更新 VLM 离散生成；但 Alpamayo 1.5 模型结构中 action expert 有 2.3B 参数，是连续控制质量的关键。把它纳入 RL 是最直接的性能上限来源。

代码落点：新增 flow RL trainer 或在 SFT stage2 trainer 后加 RL mode；复用 FlowMatching.construct_training_data() 与 compute_loss_from_pred()，加入 old/ref expert 缓存。

4Tail-Risk GRPO：按长尾危险度重塑 advantage

核心想法：不要让普通直行样本主导 advantage。对 scenario 做 risk bin：cut-in、crosswalk、intersection、adverse weather、off-road risk。组内 GRPO 改为 difficulty-aware normalization，并用 CVaR 风格的 tail reward 强化最差分位数。

为什么新：Alpamayo-R1 的公开摘要强调 long-tail 场景和 closed-loop 安全收益；当前仓库 reward 是单样本 ADE + comfort，缺少面向长尾失败的优化目标。

代码落点：在 prefetch/dataset 或 reward metadata 中输出 risk tag，trainer 中按 risk group 汇总 reward，并记录 per-risk W&B/TensorBoard 指标。

5Self-Consistency as Reward：多样本 CoC/轨迹一致性

核心想法：同一个 prompt 生成 K 条 CoC + 轨迹，不只比较 reward 高低，还计算解释一致性、轨迹 mode collapse、多模态覆盖率。对于本来存在多种合理驾驶策略的场景，奖励“安全多模态”；对于确定性场景，奖励“低方差稳定”。

为什么新：GRPO 已经天然对同 prompt 多采样；我们可以把多样本统计变成 reward，不需要额外 rollout 架构。

5. 实验矩阵

主要 baseline

1. SFT-only Alpamayo 1.5。
2. 当前 GRPO：ADE + comfort。
3. CoC reward only。
4. Flow expert supervised fine-tuning only。
5. Causal Flow-GRPO full。

核心指标

Open-loop：minADE_6、FDE、jerk、yaw rate、trajectory validity。
Reasoning：CoC judge score、format pass rate、causal consistency。
Closed-loop：off-road、collision/close encounter、route progress、AlpaSim score。

实验 验证问题 预期现象
Token advantage routing ablation 同一个 reward 是否应该推所有 token？ per-token routing 降低格式崩坏，提高 CoC-action consistency。
Counterfactual reward ablation CoC 是否只是好看的解释？ 加入 counterfactual 后，危险场景 close encounter/off-road proxy 降低。
Flow RL ablation 只训离散 VLM 是否限制连续轨迹质量？ Flow expert RL 对 comfort、FDE、closed-loop stability 更明显。
Risk-bin generalization 是否真的改善长尾？ 普通场景不退化，cut-in/intersection/pedestrian bins 提升更大。
