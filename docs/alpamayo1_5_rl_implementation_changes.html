3.2 coc_reward.py NEW

功能：CoC推理文本质量评分

从 rollout completion 中提取 Chain-of-Causation 推理文本（<|cot_end|>之前的部分），在四个维度上评分：

维度函数评分方法权重
事实准确性 score_factual_accuracy() 4类场景实体正则匹配（车辆/道路/天气/交通） 0.25
因果连贯性 score_causal_coherence() 因果连接词 + 决策关键词 + 结构检查（观察→行动） 0.30
安全意识 score_safety_awareness() 安全关键词匹配（hazard/risk/collision/cautious等） 0.25
完整性 score_completeness() 6个驾驶维度覆盖检查（危险识别/意图预测/动作论证等） 0.20

关键数据结构

# 场景实体模式 — 4类共10+正则模式
SCENE_ENTITY_PATTERNS = {
    "vehicle": ["vehicle|car|truck|bus|...", "ego vehicle|leading vehicle|..."],
    "road_structure": ["lane|intersection|crosswalk|...", "highway|ramp|..."],
    "weather": ["rain|snow|fog|sun|glare|night|wet|icy|..."],
    "traffic_condition": ["congestion|heavy traffic|light traffic|..."],
}

# 因果连接词 — 检查推理链内的逻辑连接
CAUSAL_CONNECTORS = ["because|since|due to|as a result|therefore|thus|..."]

# 6个驾驶维度 — 用于完整性评分
DRIVING_DIMENSIONS = {
    "hazard_identification", "intent_prediction",
    "action_justification", "safety_check",
    "spatial_reasoning", "temporal_reasoning",
}

CoC文本提取逻辑

def extract_coc_text(to_be_evaluated: str) -> str:
    """从rollout completion提取CoC文本。"""
    if "<|cot_end|>" in to_be_evaluated:
        return to_be_evaluated.split("<|cot_end|>")[0].strip()
    if "<|traj_future_start|>" in to_be_evaluated:
        return to_be_evaluated.split("<|traj_future_start|>")[0].strip()
    return to_be_evaluated.strip()

3.3 raa_reward.py NEW

功能：推理-动作一致性（RAA）计算

这是弥合CoC-轨迹差距的核心组件。从CoC文本中提取行为意图，从轨迹中推断实际行为，计算两者对齐度。

三步流水线

# Step 1: 从CoC文本提取行为约束
constraints = extract_behavioral_constraints(coc_text)
# → {"decelerate": 0.85, "turn_right": 0.70, "maintain": 0.3, ...}

# Step 2: 从预测轨迹推断实际行为
inferred = infer_trajectory_behavior(predicted_xyz, predicted_rot)
# → 通过速度变化推断加减速，方向变化推断转向，横向位移推断变道

# Step 3: 计算对齐度
# 对于意图行为（intended > 0.3）：alignment = 1 - |intended - actual|
# 对于非意图行为+实际高：penalty = 1 - actual

7种行为约束类型

行为CoC检测方法轨迹推断方法
减速正则匹配 slow down/decelerate/brake/stop/yield速度变化率 sigmoid(-Δv×10)
加速正则匹配 speed up/accelerate/proceed/go速度变化率 sigmoid(Δv×10)
左转正则匹配 turn left/left turn/steer left累积方向变化 sigmoid(Δθ×5)
右转正则匹配 turn right/right turn/steer right累积方向变化 sigmoid(-Δθ×5)
变道正则匹配 change lane/merge/move to横向位移 |Δy| / 3.5
保持正则匹配 maintain/keep/stay/follow/cruise1 - min(σ(v)/2, 1)
保持距离正则匹配 keep distance/safe distancemaintain × (1-max(decel,accel))

3.4 hcc_reward.py NEW

功能：四层层次化奖励聚合

HCC-RM的主入口，按四层结构聚合所有奖励信号：

层次化公式

if s1 (场景理解) < τ_scene:
    R = -1.0  # 门控失败
else:
    # 各层归一化到 [-1, 1]
    s2_norm = 2 * s2_coc - 1      # CoC质量 [0,1]→[-1,1]
    s3_norm = 2 * s3_raa - 1      # RAA对齐 [0,1]→[-1,1]
    s4_norm = w·traj + cw·comfort  # 轨迹质量 [-1,1]

    R = α·s2_norm + β·s3_norm + γ·s4_norm
    R *= s1  # 场景理解全局乘数
    R = clamp(R, -1, 1)

聚合权重分配

从TOML读取 coc_quality_weight 和 raa_weight，剩余权重自动分配给轨迹质量。

返回的reward_dict（用于日志和多目标分解）

reward_dict = {
    "scene_understanding": float,    # s1
    "coc_quality": float,            # 聚合CoC质量
    "coc_factual": float,            # 事实准确性
    "coc_coherence": float,          # 因果连贯性
    "coc_safety": float,             # 安全意识
    "coc_completeness": float,       # 完整性
    "raa_score": float,              # RAA对齐度
    "raa_decelerate": float,         # 各约束对齐度...
    "traj_L2": float,                # ADE距离
    "comfort_reward": float,         # 舒适度
    "reward": float,                 # 最终奖励
    "reward_type": "hcc",           # 或 "hcc_gated_scene"
}

Pareto距离加权增强

# 理想点 = 各维度最大值
ideal_point = objectives.max(dim=0)

# Pareto样本中离理想点近的 → 更高权重（最高可达 2×base_weight）
distances = ||(ideal_point - objectives) / range||
pareto_dist_weights = 1 / (distances + ε)

weights[pareto] = base_weight * (1 + (1 - normalized_distance))

4.3 adaptive_kl.py NEW

功能：AdaptiveKLScheduler 类

核心公式

β_t = β_base · exp(-λ · (s̄_align,t - 0.5))
  • RAA高 → β低 → 鼓励探索（策略值得信任时放宽KL约束）
  • RAA低 → β高 → 保守更新（推理-动作不一致时加强SFT锚定）

关键参数

参数默认值说明
base_beta0.04基线KL系数（RAA≈0.5时的值）
lambda_decay2.0指数衰减率，越大越敏感
min_beta / max_beta0.001 / 0.2β 取值范围
window_size100RAA历史窗口
warmup_steps50前50步使用固定base_beta
smoothing0.9EMA平滑系数

RAA信号处理

# EMA平滑
ema_raa = 0.9 * ema_raa + 0.1 * raa_score

# 窗口平均（最近50步）
window_avg = mean(raa_history[-50:])

# 混合信号
smoothed_raa = 0.5 * ema_raa + 0.5 * window_avg

4.4 contrastive_loss.py NEW

功能：对比推理-动作学习（CRAL）

  • compute_contrastive_loss() — 完整版：使用CoC质量+RAA组合分数划分正负样本
  • compute_contrastive_loss_simple() — 简化版：按reward排名划分top 50% vs bottom 50%（GSPOTrainer使用此版本）

核心机制

# 取最后一层hidden states的均值池化
pooled = hidden_states.mean(dim=1)  # [B, D]

# L2归一化
pooled = F.normalize(pooled, p=2)

# InfoNCE: 正样本对互相靠近，远离负样本
# 按reward排名：top 50% = 正样本, bottom 50% = 负样本
L_contrastive = -log(Σexp(sim_pos/τ) / (Σexp(sim_pos/τ) + Σexp(sim_neg/τ)))

这种对比损失鼓励模型学到：好的推理应该对应好的轨迹，两者在隐空间中编码为相似表示。

5.2 gspo_loss.py NEW

功能：GSPO序列级损失计算

7步计算流程

# Step 1: 聚合每token log-prob到序列级
for each sequence i in batch B:
    seq_current_logps[i] = sum(current_per_token_logps[start:end])
    seq_old_logps[i] = sum(old_per_token_logps[start:end])

# Step 2: 序列级重要性比率
log_ratio = seq_current_logps - seq_old_logps
ratio = exp(log_ratio)  # w_i

# Step 3: 序列级裁剪（关键差异！GRPO是per-token clip）
ratio_clipped = clamp(ratio, 1-ε_low, 1+ε_high)

# Step 4: 序列级损失
loss_unclipped = -ratio * advantage
loss_clipped = -ratio_clipped * advantage
gspo_per_seq = max(loss_unclipped, loss_clipped) if A>0
             = min(loss_unclipped, loss_clipped) if A≤0

# Step 5: 仅未裁剪样本归一化（关键差异！GRPO用所有token）
is_unclipped = (ratio in [1-ε, 1+ε])
gspo_loss = sum(gspo_per_seq) / count(is_unclipped)

# Step 6: KL惩罚（序列级）
kl_loss = β * mean(Σ_t KL(π_θ||π_ref) per sequence)

# Step 7: 分布式all-reduce

两个导出函数

函数用途
compute_gspo_loss()训练时的主loss计算，返回 (total_loss, per_token_loss, kl_loss)
compute_gspo_sequence_ratios()监控用，返回 (ratios, log_ratios) 用于TensorBoard日志
